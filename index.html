Projects

Monopoly

When tasked with an open ended project proposal, my group thought it would be challenging and stimulating to design a game we had spent so much of our childhood playing. The level of complexity of the game fit perfectly in our vision of an object oriented project that would result in a nostalgic and fun game!

This project allows 3-8 players to play a game of monopoly for a specified number of rounds - where the player with the highest net worth wins! Players are able to buy and sell properties, go to/get bailed out of jail, and of course collect $200,000 upon passing go. The project will come with a basic visual implementation that shows where each player is on the board.

The actual game is run by a simulator class, which contains a Board and several Players, that directs the flow of gameplay. A Board is comprised of GameSquares: LandPlots, Utilities, and CornerSquares. Players are able to buy and sell properties and charge other players rent.

In our group, I was tasked with creating the Simulator and Player classes. So I will deal with user I/O in order to get the desired actions of the player, and be able to take the correct action from each command.

AI Heart Disease Prediction Model

Worked with esteemed Data Scientist/Professor at UPenn to visualize a dataset of over 300,000 data entries from the CDC. Using machine learning models K-Nearest Neighbors, Random Forest, and XGBoost with varying features we trained and tested different models to predict heart disease In these experiments we computed accuracy metrics with and without the binary physical activity feature.

We found that the binary physical activity feature did not improve accuracy when predicting heart disease; in fact, through all three models we found a decrease in accuracy when we added the binary physical activity feature. These test results negate the notion that the binary physical activity feature improves the accuracy of heart disease prediction and calls for a more robust method of measuring physical activity.
